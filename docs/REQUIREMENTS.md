## 一、产品概述

### 1.1 产品名称

**WhyFire**

- Why = 为什么
- Fire = 火/爆款
- 含义：破解"为什么火"的密码

**Slogan**：Crack the code of viral content / 破解爆款密码

---

### 1.2 一句话定位

> **小红书短视频 0-1万粉博主的 AI 教练 — 告诉你为什么他能火，你不火**

---

### 1.3 核心价值主张

> **"告诉你为什么他能火，你不火 — 并给你看你能变得多好"**

## 二、目标用户

### 2.1 用户定义

| 维度 | 描述 |
|------|------|
| **人群** | 小红书短视频初级用户（0-10000粉博主） |
| **内容类型** | 短视频（非图文） |
| **阶段特征** | 刚起步或正在摸索期 |
| **心理状态** | 最迷茫、最焦虑、最需要指导 |

| 原因 | 说明 |
|------|------|
| **市场细分** | 小红书短视频博主，竞争相对少 |
| **人数多** | 起步博主数量远超成熟博主 |
| **被忽视** | 大厂服务品牌/MCN，起步博主付不起高价 |

## 五、技术方案详解

### 5.1 第一步：竞品分析（文本分析）

**输入：** 用户粘贴小红书视频链接

**数据获取：**

| 数据 | 获取方式 | 分析价值 |
|------|---------|---------|
| 标题 | Playwright 抓取页面 | ⭐⭐⭐⭐⭐ 最关键 |
| 正文 | Playwright 抓取页面 | ⭐⭐⭐⭐ 内容结构 |
| 话题标签 | Playwright 抓取页面 | ⭐⭐⭐ 流量入口 |
| 点赞数 | Playwright 抓取页面 | ⭐⭐⭐ 热度指标 |
| 收藏数 | Playwright 抓取页面 | ⭐⭐⭐⭐ 实用性指标 |
| 评论数 | Playwright 抓取页面 | ⭐⭐⭐ 话题性指标 |
| 封面图 | Playwright 抓取页面 | ⭐⭐⭐ 视觉分析 |

**关键指标计算：**
```
收藏/点赞比 > 0.5 → 实用性强（干货类）
收藏/点赞比 < 0.2 → 情绪驱动（共鸣类）
```

**LLM 分析输出：**
- 标题为什么有效（关键词 + 情绪触发点 + 结构）
- 内容结构拆解
- 话题标签策略
- 可复制模板

**登录方案（2026-03-16 验证通过）：**

由于小红书对未登录访问有严格限制，采用 Cookie 扫码登录方案：

```
用户流程：
1. 首次使用 → 点击"登录小红书"
2. 弹出浏览器窗口 → 显示二维码
3. 用户扫码 → 自动完成登录
4. 系统保存 Cookie → 后续请求自动使用
5. Cookie 过期 → 提示重新登录
```

**技术实现：**
- 参考 [MediaCrawler](https://github.com/NanmiCoder/MediaCrawler) 开源项目
- Playwright 打开登录页 → 等待用户扫码 → 保存 Cookie
- 后续请求注入 Cookie → 正常获取页面数据
- Cookie 有效期约 7-30 天

**技术优势：**
- ✅ 不需要下载视频
- ✅ 扫码登录，用户体验好
- ✅ 参考 mature 开源方案，技术风险低

---

### 5.2 第二步：用户视频诊断（视频分析）

**输入：** 用户上传自己的短视频

**技术方案：**
- 用户直接上传视频到服务端
- 多模态 LLM（Claude/GPT-4V）分析视频内容
- 重点分析前 3 秒的留存要素

**输出：**
- 问题诊断
- 与爆款的差距对比
- 具体改进建议

---

### 5.3 第五步：AI 检查钩子效果（视频对比）

**核心逻辑：**

只对比 3-5 秒的钩子开头，不是整个视频。

```
模板：钩子开头示例（3-5秒）─┐
                           ├─→ 多模态 LLM 对比 → 评分 + 改进建议
用户：重新拍摄的开头（3-5秒）┘
```

**目标：** 帮助用户优化前3秒，提高留存率

**技术实现：**

```
┌─────────────────────────────────────────┐
│  输入                                    │
│  ────                                    │
│  1. 模板钩子（文字脚本 + 示例视频）       │
│  2. 用户拍摄的 3-5 秒开头视频            │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  多模态 LLM（Claude/GPT-4V）             │
│  ────                                    │
│  Prompt: 这个开头能留住观众吗？          │
│          和模板相比差在哪里？            │
│          给出评分和改进建议              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  输出                                    │
│  ────                                    │
│  钩子效果评分：70/100                    │
│  改进建议：                              │
│  - 开头表情可以更困扰一点                │
│  - 语速稍微快一点效果更好                │
│  - 第一句话的语气要更坚定                │
└─────────────────────────────────────────┘
```

**技术优势：**

| 特点 | 说明 |
|------|------|
| **视频短** | 只分析 3-5 秒，处理快、成本低 |
| **目标明确** | 判断"钩子效果怎么样"，不需要复杂量化 |
| **实现简单** | 一个多模态 LLM API 调用即可 |
| **反馈直观** | 评分 + 文字建议，用户易懂 |

**固定成本（腾讯云）：**

| 项目 | 月费用 |
|------|--------|
| 轻量服务器 (4核8G) | ¥200-280 |
| 域名 + CDN | ¥30 |
| **合计** | **¥230-310** |

**盈亏平衡点：**

```
月固定成本 = ¥280
平均单用户毛利 = ¥17
盈亏平衡用户数 = 17 付费用户/月
```

**成本优化策略（可提升毛利率至 78-91%）：**

1. **Prompt Caching** — 节省 90% 重复 prompt 成本
2. **智能模型路由** — 简单请求用 Haiku，节省 50%
3. **结果缓存** — 相同链接 24h 内复用

**核心策略：**
- 免费版 = 纯文本分析（成本极低，¥0.21/月）
- 视频功能 = 付费专属（成本较高）
- Pro 版设置软上限防滥用
- 年付优惠 = 10% 折扣，改善现金流

详细商业分析见：`/docs/plans/2026-03-16-business-analysis.md`

```
WhyFire（诊断+建议）→→→ VidLuxe（工具执行）
        ↑                       ↓
        ←←← 用户口碑传播 ←←←←←←←←
```

### 10.2 协同逻辑

| 产品 | 定位 | 价值 |
|------|------|------|

| **VidLuxe** | 工具箱（执行） | 一键帮你优化封面、图片 |

- **不是强制**：给用户建议，而非强制执行
- **展示对比**：封面前后对比图，让用户自己意识到差距
- **自然导流**：用户想优化时，VidLuxe 是最方便的选择

### Playwright 页面抓取方案（2026-03-16 已验证）

**问题：** 小红书对未登录访问有严格限制，所有无 Cookie 请求都会被重定向到登录页。

**解决方案：** Cookie 扫码登录方案（参考 [MediaCrawler](https://github.com/NanmiCoder/MediaCrawler)）

**验证结果：**
- ✅ 扫码登录成功
- ✅ Cookie 保存成功（14 个关键 Cookie）
- ✅ Cookie 注入后可正常访问笔记页面
- ✅ 无头模式（headless）可正常工作

**技术实现：**
```
1. Playwright 打开小红书登录页
2. 显示二维码，等待用户扫码
3. 登录成功后保存 Cookie 到本地/服务端
4. 后续请求自动注入 Cookie
5. Cookie 有效期约 7-30 天
```

**测试脚本：** `scripts/test_qrcode_login.js`
