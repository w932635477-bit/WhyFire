## 一、产品概述

### 1.1 产品名称

**WhyFire**

- Why = 为什么
- Fire = 火/爆款
- 含义：破解"为什么火"的密码

**Slogan**：破解爆款密码，找到你的差距 / Crack the code of viral content

### 1.2 一句话定位

> **小红书 AI 教练 — 分析竞品、诊断自己、对比差距、获取灵感，一步步引导你做出爆款**

### 1.3 核心价值主张

> **"我为什么没有流量，他却有那么多流量？" — AI 帮你找到答案**

## 二、目标用户

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| **人群** | 小红书起步博主（0-1万粉） |
| **核心痛点** | "不知道为什么别人能火，自己不行" |
| **心理状态** | 认知焦虑，缺乏参照系 |
| **信息困境** | 只看到结果，不知道差距在哪 |
| **支付能力** | 有限，需要低价/免费方案 |
| **人数规模** | 最大的人群，但被大厂忽视 |

### 2.2 为什么选择这个人群？

| 原因 | 说明 |
|------|------|
| **痛点真实** | "为什么他能火"是最常问的问题 |
| **人数最多** | 起步博主数量远超成熟博主 |
| **被忽视** | 大厂服务品牌/MCN，起步博主付不起高价 |
| **可培养** | 从小白开始培养，长期价值高 |

## 三、产品形态：向导式 AI 教练

### 3.1 设计理念

```
传统工具：功能堆砌，用户自己决定用哪个
WhyFire：AI 教练，一步步引导用户完成分析流程
```

### 3.2 用户旅程

```
Step 1: 竞品分析     →  分析爆款笔记，拆解成功要素
Step 2: 内容诊断     →  分析自己内容，找出差距
Step 3: 可视化对比   →  对比差距，一目了然
Step 4: 灵感推荐     →  精选 6-10 个相关海外爆款
Step 5: 完成提醒     →  提醒用户拍完再回来诊断
```

## 四、四大功能模块

### 4.1 模块一：竞品分析

**目的**：让用户理解"为什么他能火"

**输入**：竞品/爆款笔记链接

**AI 输出**：
- 🎯 钩子强度评分（标题分析）
- 💡 内容价值评分
- 🎨 视觉吸引力评分
- 🔥 爆款潜力评分
- 📊 关键洞察（为什么能火）

### 4.2 模块二：内容诊断

**目的**：让用户知道"我的差距在哪"

**输入**：用户自己的笔记链接 + Step 1 的竞品数据

**AI 输出**：
- 综合评分
- 各维度与竞品的差距
- 具体改进建议（高/中/低优先级）

### 4.3 模块三：可视化对比

**目的**：让差距一目了然

**展示**：
- 雷达图：你的内容 vs 竞品
- 差距柱状图：具体差距数值
- 核心差距总结：最大的 3 个差距点
- 优先改进建议

### 4.4 模块四：海外创意灵感库

**目的**：给用户可借鉴的创作灵感

**设计原则**：
- ⚠️ **精准推荐，不是瀑布流浏览**
- 每次推送 **6-10 个**精选视频
- 基于前几步的分析结果，智能匹配相关内容
- 每个视频附带：创意解读、本土化建议、可借鉴元素

**数据源**：
- YouTube Data API（主数据源）
- TikTok via Apify（辅助数据源）

## 五、技术方案

### 5.1 技术架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│  前端层                                                                  │
│  └── Next.js 15 + React 18 + Tailwind CSS                              │
│      → 向导式 UI + 进度条 + 步骤导航                                     │
├─────────────────────────────────────────────────────────────────────────┤
│  API 层                                                                  │
│  ├── /api/analyze/competitor   → 竞品分析                               │
│  ├── /api/analyze/self         → 内容诊断                               │
│  ├── /api/compare              → 可视化对比                             │
│  └── /api/inspiration          → 灵感推荐                               │
├─────────────────────────────────────────────────────────────────────────┤
│  数据源层                                                                │
│  ├── 小红书公开数据（Playwright）→ 竞品分析、内容诊断                     │
│  ├── YouTube Data API           → 灵感推荐                              │
│  └── TikTok via Apify           → 灵感推荐（辅助）                       │
├─────────────────────────────────────────────────────────────────────────┤
│  AI 层                                                                   │
│  ├── Claude Sonnet → 竞品分析、内容诊断（深度分析）                       │
│  └── Claude Haiku  → 灵感解读（低成本）                                  │
├─────────────────────────────────────────────────────────────────────────┤
│  存储层                                                                  │
│  └── Session Storage (MVP) → SQLite (V2)                               │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 核心依赖

| 依赖 | 用途 | 成本 |
|------|------|------|
| **小红书公开数据** | 竞品/自己笔记分析 | 免费 |
| **YouTube Data API** | 海外灵感推荐 | 免费（10,000 单位/天） |
| **Apify TikTok** | 海外灵感推荐（辅助） | $49/月 |
| **Claude Sonnet** | 深度分析 | ~$3/1M tokens |
| **Claude Haiku** | 灵感解读 | ~$0.25/1M tokens |

**月成本估算：约 ¥500/月**

## 八、差异化定位

### 8.1 vs 商业竞品

| 维度 | 千瓜/新红 | WhyFire |
|------|----------|---------|
| 目标用户 | 品牌/MCN | 0-1万粉起步博主 |
| 价格 | 年费数千~数万 | Freemium |
| 核心价值 | 数据报表 | AI 教练（告诉你该干嘛） |
| 使用体验 | 功能复杂 | 向导式，5 分钟上手 |
| 数据来源 | 国内平台 | 国内 + 海外（差异化） |

### 8.2 核心竞争力

```
1. 向导式流程：一步步引导，不需要用户思考
2. AI 教练定位：不是冷冰冰的工具，是教练
3. 精准灵感推荐：6-10 个精选，不是海量浏览
4. 价格策略：起步博主付得起
5. 海内外结合：国内分析 + 海外灵感
```
