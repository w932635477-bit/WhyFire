# WhyFire v2.0 MVP 技术决策设计文档

**日期**: 2026-03-18
**状态**: Approved
**版本**: 1.0

---

## 1. 背景

在实现 WhyFire v2.0 PRD 之前，需要对几个关键技术决策进行明确，以确定 MVP 阶段的技术范围和实施方案。

---

## 2. 技术决策

### 2.1 FFmpeg.wasm 方案

**决策：纯浏览器端方案**

| 项目 | 说明 |
|------|------|
| 方案 | 所有视频合成在用户浏览器端通过 FFmpeg.wasm 完成 |
| 低端设备处理 | 检测设备性能，低端设备显示"建议使用电脑获得更好体验"提示 |
| 服务端方案 | 不实现服务端备选方案 |

**理由**：
- MVP 阶段目标是验证产品价值，不是追求完美体验
- 目标用户（电商卖家、内容创作者）大多有电脑
- 添加服务端方案会增加开发时间和运维成本
- 如果用户反馈强烈，可以在 Beta 阶段添加服务端方案

---

### 2.2 方言 TTS 方案

**决策：全部使用 MiniMax TTS + Prompt 工程**

| 方言 | 实现方式 |
|------|----------|
| 普通话 | MiniMax TTS（原生支持） |
| 东北话 | MiniMax TTS + 方言 Prompt 引导口音 |
| 粤语 | MiniMax TTS + 方言 Prompt 引导口音（先实测效果） |
| 四川话 | MiniMax TTS + 方言 Prompt 引导口音 |

**备选方案**：
- 如果粤语效果不理想，考虑 Azure 粤语 TTS 或延后粤语支持

**理由**：
- 单一服务商降低对接复杂度
- MiniMax TTS 对各方言支持程度需要实测验证
- Prompt 工程可以引导生成带有方言特色的语音

---

### 2.3 动感字幕方案

**决策：纯浏览器端 Canvas 渲染 + FFmpeg overlay**

| 项目 | 说明 |
|------|------|
| 渲染方式 | Canvas 2D API 实时渲染字幕动画 |
| 同步机制 | requestAnimationFrame 同步歌词时间轴 |
| 合成方式 | FFmpeg.wasm 将 Canvas 帧序列 overlay 到视频轨道 |

**MVP 字幕动效**：
- 基础弹跳动效（1种）

**理由**：
- 动感字幕是核心差异化功能，MVP 应保留
- 与 FFmpeg.wasm 方案一致，保持浏览器端处理
- 字幕渲染比视频合成轻量，普通设备可以处理
- 30秒视频的字幕渲染，普通设备完全可以处理

---

### 2.4 MVP 支付方案

**决策：MVP 阶段不实现支付功能**

| 项目 | 说明 |
|------|------|
| 新用户体验 | 注册赠送 5 积分（约 2.5 个视频） |
| 积分用完 | 引导用户等待 Beta 版本 |
| 支付上线时间 | Beta 阶段（Week 6-7） |

**理由**：
- MVP 目标是验证"用户是否愿意用这个产品生成视频"，不是验证付费
- 5 积分足够用户完成完整流程体验
- 支付系统（尤其是微信支付）涉及商户申请、合规、测试，工作量大
- 如果产品本身没人用，做支付也是白费

---

### 2.5 Hero 动态背景视频

**决策：使用 AI 生成 Rap 视频素材**

| 项目 | 说明 |
|------|------|
| 素材来源 | Sora / Runway / Kling 等 AI 视频生成工具 |
| 视频规格 | 1920x1080, 8-10秒循环, <5MB |
| 替换时机 | 产品上线后可用真实 Demo 录屏替换 |

**制作建议**：
- 提前准备，在 MVP 开发期间同步生成素材
- 生成 2-3 个精彩 Rap 视频片段
- 后期加速播放 + 暗色遮罩处理

## 3. MVP 技术范围确认

### 3.1 保留功能

| 功能 | 技术方案 | 优先级 |
|------|----------|--------|
| 四大场景选择 | React 组件 | P0 |
| AI 对话式歌词创作 | Claude API | P0 |
| 方言支持（4种） | MiniMax TTS + Prompt | P0 |
| 音乐生成 | MiniMax API（异步） | P0 |
| 动感字幕 | Canvas + FFmpeg overlay | P0 |
| 视频合成 | FFmpeg.wasm（浏览器端） | P0 |
| 邮箱验证码登录 | Supabase Auth | P0 |
| 新用户赠送5积分 | Supabase + 积分逻辑 | P0 |
| Hero 动态背景 | AI 生成视频素材 | P0 |
| 视频导出 | 720p + 水印 | P0 |

### 3.2 延后功能

| 功能 | 延后到 | 原因 |
|------|--------|------|
| 微信支付 | Beta | MVP 不需要支付验证 |
| 订阅功能 | Release | 依赖支付系统 |
| 1080p 无水印 | Alpha/Beta | 服务器带宽考虑 |
| 视频特效/滤镜 | Beta | 功能优先级较低 |
| 模板库 | Beta | 依赖素材积累 |

---

## 4. 技术栈确认

```
Frontend:  Next.js 15 + React 18 + TypeScript + Tailwind CSS
AI APIs:   Claude API (歌词+对话) + MiniMax API (音乐+TTS)
Video:     FFmpeg.wasm (浏览器端视频合成)
字幕:      Canvas 2D API + FFmpeg overlay
特效:      CSS Animations
Hosting:   Vercel
Database:  Supabase (PostgreSQL + Auth)
Cache:     Upstash Redis (任务队列 + 限流 + 缓存)
Storage:   Cloudflare R2 (文件存储)
```

---

## 5. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| MiniMax 粤语效果不理想 | 准备 Azure TTS 备选方案，或延后粤语支持 |
| FFmpeg.wasm 低端设备性能差 | 显示明确提示，建议使用电脑 |
| AI 生成背景视频质量不稳定 | 多生成几个版本，择优使用 |
| MVP 用户用完5积分流失 | 收集邮箱，Beta 上线时通知 |
