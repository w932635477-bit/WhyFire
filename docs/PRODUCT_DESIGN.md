## 核心价值主张

**"告诉你为什么他能火，你不火 — 并给你看你能变得多好"**

## 一句话定位

**小红书 0-1万粉博主的 AI 教练 — 诊断问题、对比差距、给出可执行建议**

---

## 一、产品名称

**WhyFire**

- Why = 为什么
- Fire = 火/爆款
- 含义：破解"为什么火"的密码

Slogan：**Crack the code of viral content** / 破解爆款密码

## 三、核心洞察：博主的四种焦虑

| 焦虑类型 | 描述 | 严重程度 |
|---------|------|---------|
| **方向焦虑** | "不知道发什么" | ⭐⭐⭐⭐⭐ |
| **认知焦虑** | "不知道为什么别人能火" | ⭐⭐⭐⭐⭐ |
| **对标焦虑** | "不知道学谁" | ⭐⭐⭐⭐ |
| **效果焦虑** | "发了没人看" | ⭐⭐⭐⭐ |

**核心问题**：
> "我的视频为什么没有流量？竞争对手的流量是怎么来的？为什么他会火，而我不行？"

## 四、差异化定位

### vs 开源工具（redbook）

| 维度 | redbook | WhyFire |
|------|---------|---------|
| 用户 | 开发者 | 普通博主 |
| 形态 | CLI 命令行 | Web 产品 |
| 价值 | 数据 API | AI 解读 + 行动建议 |
| 门槛 | 需技术能力 | 打开即用 |

### vs 商业竞品（千瓜/新红）

| 维度 | 千瓜/新红 | WhyFire |
|------|----------|---------|
| 目标用户 | 品牌/MCN | 0-1万粉起步博主 |
| 价格 | 年费数千~数万 | Freemium（免费+¥29/月） |
| 核心价值 | 数据报表 | AI 教练（告诉你该干嘛） |
| 学习成本 | 需要培训 | 5分钟上手 |

### 核心竞争力

```
1. 用户定位：被大厂忽视的起步博主（人数最多）
2. 价格策略：他们付得起的低价
3. 核心价值：不是数据，是"答案"
4. 情感定位：AI 教练，不是冷冰冰的工具
5. 使用体验：零门槛，打开即用
```

## 五、核心功能

### 功能定位

**帮用户分析竞争对手，分析对手的内容和细节，与用户做对比分析，给出解决方案**

### 用户粘性闭环

```
用户来咨询 → 诊断问题 → 看到对比图 → 发现"原来可以这样"
    ↑                                    ↓
    ←←←←← 下次还想来 ←←←←←←←←←←←←←←←←←←←←
```

**为什么用户会回来？**
- 每次都有新发现
- 每次都能看到"更好的自己"
- 形成习惯：有困惑 → 问 WhyFire

### MVP 功能（第一版）

```
┌─────────────────────────────────────────────────────────┐
│  功能 1：爆款拆解                                        │
│  ─────────────────────────────────────                  │
│  输入：用户粘贴爆款笔记链接                               │
│  输出：                                                  │
│  - 标题为什么有效（关键词 + 情绪触发点 + 结构）             │
│  - 内容结构拆解（开头/中间/结尾的模式）                    │
│  - 话题标签策略（为什么选这些标签）                        │
│  - 互动数据解读（收藏/点赞比 = 实用性）                    │
│  - 可复制模板                                            │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  功能 2：我的内容诊断                                    │
│  ─────────────────────────────────────                  │
│  输入：用户自己的笔记链接                                 │
│  输出：                                                  │
│  - 标题评分 + 优化建议                                    │
│  - 内容问题诊断                                          │
│  - 对比爆款差距在哪里                                     │
│  - 具体修改建议                                          │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  功能 3：可视化对比（核心差异化）                         │
│  ─────────────────────────────────────                  │
│  输入：用户笔记 + 优化建议                                │
│  输出：                                                  │
│  - 封面优化前后对比图                                     │
│  - 标题优化前后对比                                       │
│  - "给你看你能变得多好"                                   │
│  - 软性引导：不是强制，让用户自己意识到差距                │
│  - 配合 VidLuxe 提供一键优化（可选）                      │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  功能 4：选题灵感生成                                    │
│  ─────────────────────────────────────                  │
│  输入：用户领域 + 3-5 个爆款参考                          │
│  输出：                                                  │
│  - 基于爆款模式的选题建议                                 │
│  - 标题模板生成                                          │
│  - 内容框架建议                                          │
└─────────────────────────────────────────────────────────┘
```

## 六、商业协同

### WhyFire + VidLuxe 产品矩阵

```
WhyFire（诊断+建议）→→→ VidLuxe（工具执行）
        ↑                       ↓
        ←←← 用户口碑传播 ←←←←←←←←
```

### 协同逻辑

| 产品 | 定位 | 价值 |
|------|------|------|
| **WhyFire** | AI 教练（诊断） | 告诉你问题在哪，你能变得多好 |
| **VidLuxe** | 工具箱（执行） | 一键帮你优化封面、图片 |

### 软性引导策略

- **不是强制**：给用户建议，而非强制执行
- **展示对比**：封面前后对比图，让用户自己意识到差距
- **自然导流**：用户想优化时，VidLuxe 是最方便的选择

## 七、技术方案

### 数据获取：低风险方案

```
技术实现：
- Playwright/Puppeteer 访问笔记页面（无需登录）
- 只获取公开可见的信息
- 不需要 Cookie，不需要认证
- 发送给 LLM 做分析
```

### 可获取的公开数据

| 数据 | 分析价值 |
|------|---------|
| 标题 | ⭐⭐⭐⭐⭐ 最关键 |
| 正文 | ⭐⭐⭐⭐ 内容结构分析 |
| 话题标签 | ⭐⭐⭐ 流量入口 |
| 点赞数 | ⭐⭐⭐ 热度指标 |
| 收藏数 | ⭐⭐⭐⭐ 实用性指标 |
| 评论数 | ⭐⭐⭐ 话题性指标 |
| 发布时间 | ⭐⭐ 发布策略 |
| 封面图 | ⭐⭐⭐ 视觉分析 |

### 关键指标：收藏/点赞比

```
收藏/点赞比 > 0.5 → 实用性强（干货类）
收藏/点赞比 < 0.2 → 情绪驱动（共鸣类）
```

## 七、信任建立策略

### 信任金字塔

```
第一层：零门槛体验
- 完全免费试用，无需注册
- 粘贴链接 → 立即看到结果
- 先给价值，再谈付费

第二层：真实案例展示
- 展示真实爆款的分析报告
- "这条笔记为什么有 10 万赞"
- 用真实数据证明 AI 分析的价值

第三层：透明度
- 告诉用户 AI 如何分析（不是黑盒）
- 展示分析依据："标题得分 85 分，因为..."
- 让用户能验证结果的合理性

第四层：隐私承诺
- 明确声明：不存储用户笔记内容
- 只分析公开信息，不触碰私密
- 可选：提供"分析后立即删除"选项

第五层：社会认同
- 用户评价/反馈展示
- "xxx 博主说有用"
- 累计分析 xx 条笔记
```
