# 方言Rap生成系统 - 需求文档

## 一、项目概述

### 1.1 项目背景
打造一款能够生成爆款短视频的方言Rap生成系统。**核心创新**：用户用自己的声音，唱出方言Rap，歌词结合热点话题和个人特色，让用户有极强的参与感和传播欲望。

### 1.2 目标用户
- 短视频创作者（抖音/快手/B站）
- 方言文化爱好者
- 想要创作个性化内容的普通用户
- 自媒体运营者

### 1.3 产品定位（✅ 2026-03-25 确认）

**核心定位：方言 Rap 卡拉OK**

| **是** | **不是** |
|--------|----------|
| 高传播性短视频工具 | 专业 AI 音乐生成器 |
| "这是我唱的"感觉 | 唱片级音质 |
| 有趣、简单、适合分享 | 专业 Rap 制作 |
| 固定 BGM + 用户声音 | Suno/Udio 式随机生成 |

**传播性 > 音乐品质**

> 用户听到生成的 Rap 时，反应应该是"卧槽！这是我唱的？！"，而不是"这歌写得真好"。

### 1.4 核心价值
- **🎤 声音个性化**：用户用自己的声音唱Rap，参与感极强
- **📝 内容个性化**：歌词结合热点/热梗/节日 + 用户自我描述
- **🌍 文化趣味**：方言演绎增加趣味性和传播性
- **🔥 爆款潜力**：自己的声音 + 热点话题 + 方言Rap = 高传播

## 二、功能需求

### 2.1 核心功能

#### F0 - 用户声音克隆（🔥核心差异化功能）

**功能描述**：用户上传自己的声音样本，系统克隆用户音色，最终生成的Rap使用用户自己的声音。

**流程**：
```
用户录音(30秒-2分钟) → 声音特征提取 → 音色模型 → 应用于方言Rap
```

**要求**：
- **录音时长**：最少30秒，建议1-2分钟
- **录音内容**：用户朗读指定文本（或自由说话）
- **录音质量**：安静环境，清晰人声
- **克隆效果**：相似度≥80%，保持用户声音特色

**用户体验**：
> "听到自己的声音唱出方言Rap，太神奇了！"

#### F2 - 个性化歌词生成（🔥核心差异化功能）

**输入组合**：
```
┌─────────────────┐
│   热点话题      │ ← 自动抓取微博/抖音热搜
├─────────────────┤
│   热梗/流行语   │ ← 如"yyds"、"绝绝子"、"泰酷辣"
├─────────────────┤
│   节日相关      │ ← 根据日期自动匹配（春节/情人节/等）
├─────────────────┤
│   用户自我描述  │ ← 用户输入：职业/爱好/性格/想说的
└─────────────────┘
          ↓
    LLM 智能生成
          ↓
    个性化Rap歌词
```

**示例**：
> 用户输入：程序员，喜欢打游戏，最近在减肥
> 热点话题：996工作制
> 生成歌词（东北话版）：
> "哎呀妈呀996真够呛，代码敲到手抽筋，
> 下班只想打游戏，减肥的事儿明天说..."

---

#### F3 - Beat管理
- **上传Beat**：用户上传自己的背景音乐
- **Beat库**：预设的Beat库供选择（分类：激情/搞笑/抒情/等）
- **格式支持**：MP3, WAV
- **BPM分析**：自动检测Beat的节奏速度

#### F4 - 方言语音合成
- **TTS引擎**：Qwen-TTS + CosyVoice 双引擎路由
- **技术方案**：
  - **CosyVoice**：粤语（龙嘉欣）、东北话（龙老铁）- 专门音色效果更地道
  - **Qwen-TTS**：普通话（Cherry）、四川话（Sunny）、上海话（Jada）、陕西话（Marcus）、闽南语（Roy）、天津话（Peter）、南京话（Li）
- **模型**：
  - CosyVoice: cosyvoice-v3-flash
  - Qwen-TTS: qwen3-tts-flash
- **方言路由映射**：
  | 方言 | 提供商 | 模型 | 音色 |
  |------|--------|------|------|
  | 普通话 | Qwen-TTS | qwen3-tts-flash | Cherry |
  | 粤语 | CosyVoice | cosyvoice-v3-flash | longjiaxin_v3 |
  | 四川话 | Qwen-TTS | qwen3-tts-flash | Sunny |
  | 东北话 | CosyVoice | cosyvoice-v3-flash | longlaotie_v3 |
  | 上海话 | Qwen-TTS | qwen3-tts-flash | Jada |
  | 陕西话 | Qwen-TTS | qwen3-tts-flash | Marcus |
  | 闽南语 | Qwen-TTS | qwen3-tts-flash | Roy |
  | 天津话 | Qwen-TTS | qwen3-tts-flash | Peter |
  | 南京话 | Qwen-TTS | qwen3-tts-flash | Li |
- **质量要求**：原生方言音色，发音地道

#### F5 - Rhythm Adaptor（自研核心）
- **音节切分**：将连续语音切分为独立音节
- **节奏对齐**：根据BPM将音节对齐到拍子
- **时间拉伸**：高质量时间拉伸，保持音质
- **人性化处理**：添加微偏移，避免机械感

#### F6 - 混音合成
- **人声处理**：添加适量混响、压缩
- **音量平衡**：人声与Beat音量平衡
- **格式输出**：MP3（320kbps）或 WAV

### 2.2 辅助功能

#### F7 - 歌词编辑
- 手动编辑生成的歌词
- 预览歌词效果
- 保存歌词草稿

#### F8 - 效果预览
- 快速预览生成效果
- 对比不同方言版本

#### F9 - 导出分享
- 导出音频文件
- 生成分享链接

## 四、用户流程

### 4.1 首次使用流程（需要声音克隆）

```
1. 录制声音样本
   ├── 朗读指定文本（30秒-2分钟）
   └── 或上传已有录音
          ↓
2. 系统分析音色特征
   └── 显示"正在学习你的声音..."
          ↓
3. 声音模型创建完成 ✓
   └── 可随时重新录制优化
```

### 4.2 创作流程

```
1. 选择方言
   └── 普通话/粤语/四川话/东北话/上海话/陕西话/闽南语/天津话/南京话
          ↓
2. 输入个人描述
   ├── 职业/身份（如：程序员、学生、宝妈）
   ├── 爱好/特点（如：爱打游戏、减肥中、社恐）
   └── 想表达的内容（如：吐槽工作、表白、自嘲）
          ↓
3. 选择热点/话题
   ├── 系统推荐当日热搜话题
   ├── 节日自动匹配（春节/情人节/等）
   └── 用户自定义话题
          ↓
4. 生成歌词
   └── AI结合以上信息生成个性化歌词
          ↓
5. [可选] 编辑歌词
   └── 微调内容
          ↓
6. 选择Beat
   ├── 从Beat库选择
   └── 或上传自己的Beat
          ↓
7. 生成方言Rap
   ├── 用你的声音 + 选择的方言
   ├── 自动匹配节奏
   └── 显示进度
          ↓
8. 预览效果
   └── 听到"自己的声音"唱方言Rap！
          ↓
9. [可选] 调整参数
   ├── 语速/节奏调整
   └── 重新生成
          ↓
10. 导出音频
    └── 下载/分享到社交平台
```

### 4.3 用户心理体验

```
录制声音 → "这是在干什么？" → 好奇
    ↓
选择方言 → "用我的声音唱东北话？" → 期待
    ↓
输入描述 → "写关于我自己的内容" → 参与感
    ↓
生成歌词 → "这歌词说的就是我！" → 认同感
    ↓
听到结果 → "卧槽！这是我唱的？！" → 惊喜！
    ↓
分享欲望 → "我要发朋友圈/抖音" → 传播！
```

## 六、决策记录（已确认）

### 6.1 声音克隆策略（✅已验证）
- **方案**：用户说普通话录音 → CosyVoice 声音复刻 → 复刻音色 + instruction(方言) → 方言语音
- **理由**：统一使用 CosyVoice API，无需自部署，一次录音全方言通用
- **技术路径**：CosyVoice Voice Cloning API + CosyVoice TTS API
- **验证状态**：✅ 已完成技术验证
  - 声音复刻 API 可用
  - 复刻音色 + 方言指令 可用（普通话/四川话/粤语/东北话均已验证）
- **API 端点**：
  - 声音复刻：`POST https://dashscope.aliyuncs.com/api/v1/services/audio/tts/customization`
  - TTS WebSocket：`wss://dashscope.aliyuncs.com/api-ws/v1/inference/`

### 6.2 声音采集方式（✅第三轮确认）
- **方案**：用户自选二选一
  - **方式A：上传视频** - 已有视频直接传，系统自动提取人声
  - **方式B：录音朗读** - 朗读指定文字，效果更可控
- **技术要求**：
  - 时长：≥1分钟
  - 视频：需无背景音乐/噪音，人声清晰
  - 录音：需安静环境
- **视频处理流程**：上传 → 提取音轨 → 人声分离(Spleeter/Demucs) → 声音克隆
- **质量检测**：提示用户"人声是否清晰"

### 6.3 歌词风格策略（✅第二轮确认）
- **风格**：搞笑为主（固定，不需要用户选择）
- **输入组成**：
  1. 用户自己描述（职业/爱好/想说的）
  2. 热梗（yyds、绝绝子、泰酷辣...）
  3. 热点话题（当天热搜，用户可选）
  4. 节日热词（根据日期自动匹配）
- **简化**：去掉风格选择步骤，默认搞笑

### 6.4 热点话题抓取
- **MVP阶段**：只接入微博热搜Top 50
- **敏感词过滤**：建立黑名单（政治、色情、暴力等）
- **后续扩展**：抖音/B站热搜

### 6.5 Beat库规划

## 七、约束条件

### 7.1 技术约束
- **音乐控制**：必须使用用户指定的Beat，不能随机生成音乐
- **方言覆盖**：必须支持全部9种原生方言
- **云端API**：使用阿里云 Qwen-TTS + CosyVoice 双引擎
- **声音克隆**：普通话录音 → 方言输出

### 7.2 Rap 生成技术方案（✅ 2026-03-25 YC Office Hours 确认）

#### 问题诊断

当前方案 (TTS + Rhythm Adaptor) 的核心问题：
- **TTS 生成的是"读"的语音**，不是"唱"的语音
- **Rhythm Adaptor 只是在"读"的基础上调整时间**，但本质还是"读"
- 用户反馈："用改变重音的方式读歌词" ≠ "说唱"

#### 技术路线对比

| 方案 | 描述 | 优点 | 缺点 | 决策 |
|------|------|------|------|------|
| **A: AI生成 + RVC** | Suno 生成 Rap → RVC 转换音色 | 效果最可靠 | 不能固定 BGM，方言有限 | 备选 |
| **B: TTS + 演唱感增强** | 保持现有架构，增强演唱感 | 流程简单 | 技术不成熟 | 主线探索 |
| **C: 歌声合成 (SVS)** | DiffSinger 等系统 | 专业演唱 | 需要乐谱，复杂 | 暂不考虑 |

#### 决策：主线方案 B + 备选方案 A

**主线：TTS + 演唱感增强**

```
用户录音 → CosyVoice 声音克隆 → 方言 TTS → 演唱感增强 → Rhythm Adaptor → BGM 混音
                                          ↑
                                    核心攻关点
```

**演唱感增强可能的技术**：
1. **音调曲线控制** - 添加旋律起伏（flow 感）
2. **押韵字加重** - 让 punch 更明显
3. **能量包络调整** - 模拟演唱的能量变化
4. **停顿/气口处理** - 乐句之间的呼吸感

**备选：如果主线方案效果不达标，切换到 Route A (Suno + RVC)**
