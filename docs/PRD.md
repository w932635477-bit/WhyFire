## 1. 问题陈述

### 1.1 当前情况

短视频内容创作者面临以下挑战：

- **制作成本高**: 传统视频制作需要专业设备、摄影师、剪辑师，单条视频成本可达数千元
- **技术门槛高**: 视频剪辑、音乐制作、歌词创作都需要专业技能
- **制作周期长**: 从创意到成品通常需要 3-7 天
- **创意瓶颈**: 持续输出高质量创意内容困难
- **方言内容稀缺**: 缺乏支持方言（东北话、粤语、四川话等）的 AI 内容生成工具

### 1.2 用户痛点

| 痛点 | 目标用户 | 严重程度 |
|------|----------|----------|
| 视频制作太贵 | 电商卖家、小企业主 | 高 |
| 不会剪辑视频 | 内容创作者、博主 | 高 |
| 缺乏音乐创作能力 | 普通用户 | 中 |
| 创意灵感枯竭 | 专业创作者 | 中 |
| 想要方言内容但不会写 | 方言爱好者、区域创作者 | 高 |

### 1.3 商业影响

- 短视频市场规模持续增长，2025 年预计超过 3000 亿元
- AI 内容生成工具市场年增长率超过 40%
- 方言内容市场被严重低估，区域用户粘性极高
- 目标用户群体（电商卖家、内容创作者）付费意愿强

## 2. 解决方案概述

### 2.1 产品定位

**WhyFire** 是一款 AI 驱动的 Rap 视频生成工具，让用户通过简单输入即可生成专业的创意短视频。

**核心差异化**：
- 支持多方言 Rap（东北话、粤语、四川话）
- AI 对话式歌词创作，用户可与 AI 实时互动
- 动感特效 + 同步字幕，专业级视频输出

### 2.2 核心价值主张

```
对话创作 → AI 生成 Rap 歌词（含方言）→ AI 生成 Rap 音乐 → 合成特效视频
```

**四步完成，分钟级产出，方言+特效加持**

### 2.3 四大使用场景

| 场景 | 描述 | 目标用户 |
|------|------|----------|
| 产品推广 | 为产品创建 Rap 推广视频 | 电商卖家、品牌方 |
| 搞笑洗脑 | 制作搞笑、病毒式传播视频 | 内容创作者、营销人员 |
| IP 混剪 | 结合热门 IP 制作混剪视频 | 二次创作者、粉丝 |
| 日常 Vlog | 为日常生活添加 Rap 元素 | 个人用户、博主 |

#### 3.1.4.1 音乐生成异步流程

由于 MiniMax 音乐生成需要 15-45 秒，采用异步模式：

```typescript
// 流程设计
1. 用户点击"生成音乐"
2. POST /api/music/generate → 返回 { task_id, status: "pending" }
3. 前端每 3 秒轮询 GET /api/music/status/:task_id
4. 状态变化：pending → processing → completed
5. 完成后返回音乐 URL，支持在线试听

// 前端体验
- 显示进度条 + 预计等待时间（"预计 20-40 秒"）
- 支持"后台生成，完成后通知我"
```

**API 响应示例**：
```json
// 创建任务
POST /api/music/generate
Response: { "task_id": "xxx", "status": "pending", "estimated_time": 30 }

// 轮询状态
GET /api/music/status/xxx
Response: {
  "status": "processing",
  "progress": 60,
  "message": "正在生成伴奏..."
}

// 完成
GET /api/music/status/xxx
Response: {
  "status": "completed",
  "music_url": "https://...",
  "duration": 30
}
```

#### 3.1.4.2 MiniMax API 技术说明

| 功能 | 支持情况 | 说明 |
|------|----------|------|
| **TTS 语音合成** | ✅ 支持 | 支持多种音色，可通过 SSML 调整语速/停顿 |
| **音乐生成** | ⚠️ 需测试 | 有 `music_generation` API，Rap 风格效果需实测 |
| **异步模式** | ✅ 支持 | 长任务支持异步 + 轮询 |

**TTS 节奏调整**：
- MiniMax TTS 支持 SSML 标签，可使用 `<prosody rate="fast">` 加快语速
- Rap 节奏感需要后期音频处理或多次调优

**备选方案**（如果 MiniMax 效果不理想）：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Suno API** | 音乐质量更好 | 成本更高，约 $0.5/首 |
| **拆分方案** | TTS 用 MiniMax，伴奏用预置音乐库 | 灵活性降低 |
| **MusicGen 开源** | 免费部署 | 需要服务器，维护成本高 |

#### 3.2.1.1 数据库选型说明

| 组件 | 选择 | 理由 |
|------|------|------|
| **PostgreSQL** | Supabase | 免费额度大（500MB）、内置 Auth 可复用、有 GUI 管理界面、实时订阅功能可用于进度通知 |
| **Redis** | Upstash | Serverless 模式、与 Vercel 原生集成、按请求计费、免费额度足够初期使用 |

**成本预估**：
- Supabase Free：$0/月（500MB 数据库 + 1GB 文件存储 + 50k 月活用户）
- Upstash Free：$0/月（10k 请求/天）
- 初期完全免费，用户量上来后再升级 Pro 版

#### 3.2.2 方言技术方案

| 方言 | 歌词生成 | TTS 人声 | 技术方案 |
|------|----------|----------|----------|
| 普通话 | Claude API | MiniMax TTS | 原生支持，效果最佳 |
| 东北话 | Claude API + 方言 Prompt | MiniMax TTS + 风格提示词 | Prompt 工程引导口音 |
| 粤语 | Claude API + 粤语 Prompt | **Azure 粤语 TTS** | Azure 内置粤语模型，无需额外语料 |
| 四川话 | Claude API + 方言 Prompt | MiniMax TTS + 风格提示词 | Prompt 工程引导口音 |

> **技术说明**：MiniMax TTS 对粤语支持有限，因此粤语采用 Azure Cognitive Services 的粤语语音模型。Azure 已内置成熟的粤语语音，无需准备额外语料库。

#### 3.2.3 动感字幕技术方案

```
方案 A: FFmpeg + ASS 字幕（复杂但效果好，服务端渲染）
方案 B: Canvas 实时渲染 + Web 录制（灵活度高）
方案 C: Canvas 渲染 + FFmpeg overlay 合成（开发快 + 效果可控）

MVP 采用方案 C，具体实现：
1. 前端用 Canvas 2D API 实时渲染字幕动画（带跳动、渐变等特效）
2. 使用 requestAnimationFrame 同步歌词时间轴
3. 逐帧截取 Canvas 内容生成图片序列
4. FFmpeg.wasm 将图片序列 overlay 到视频轨道

后续升级到方案 A：服务端 FFmpeg + ASS 高级字幕（支持更复杂动效）
```

#### 3.2.4.1 并发与异步处理

**并发定义**：500 个用户同时在线浏览/操作（非同时生成视频）

**视频合成异步架构**：
```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  用户请求  │ ──► │ 创建任务  │ ──► │ 队列处理  │
└──────────┘     └──────────┘     └──────────┘
                       │
                       ▼
                 ┌──────────┐

                 └──────────┘
```

**超时处理**：
- 超过 90s 自动提示"合成时间较长，稍后通知下载"
- 用户可选择邮箱/微信通知

| 设备 | 30秒视频合成时间 |

## 9. 成本分析

### 9.1 API 成本详情

#### Claude API 成本（歌词生成 + AI 对话）

| 模型 | Input | Output |
|------|-------|--------|
| Claude Haiku | $0.25/1M tokens | $1.25/1M tokens |
| Claude Sonnet | $3.00/1M tokens | $15.00/1M tokens |

**每个视频消耗**：
- 对话交互：~3 轮对话
- Input tokens：~2000（用户输入 + 历史上下文）
- Output tokens：~1000（歌词 + 修改）

**推荐使用 Haiku**（成本更低，歌词生成足够）：
```
每视频 Claude 成本 = (2000 × $0.25/1M) + (1000 × $1.25/1M)
                   = $0.0005 + $0.00125
                   = $0.00175 ≈ ¥0.013
```

#### MiniMax API 成本（TTS + 音乐生成）

| 服务 | 价格 | 说明 |
|------|------|------|
| TTS 语音合成 | ~¥0.02/千字符 | 30秒歌词约150字 |
| 音乐生成 | ~¥0.5-1.0/次 | 30秒背景音乐 |

**每个视频消耗**：
```
TTS 成本：150字 × ¥0.02/1000 = ¥0.003
音乐生成成本：¥0.5-1.0/次（取平均 ¥0.75）

每视频 MiniMax 成本 = ¥0.003 + ¥0.75 ≈ ¥0.75
```
