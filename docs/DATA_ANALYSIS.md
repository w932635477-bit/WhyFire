## 一、数据全景图

### 1.1 博主维度的数据

| 数据类别 | 具体指标 | 获取难度 | 对起步博主的价值 |
|---------|---------|---------|-----------------|
| **基础画像** | 粉丝数、获赞数、笔记数、认证信息 | 低（公开） | 了解竞品规模 |
| **成长轨迹** | 涨粉趋势、周/月涨粉量 | 中（需持续监控） | 判断账号健康度 |
| **内容分布** | 内容类型比例、发布频率、发布时间规律 | 中 | 学习发布策略 |
| **爆款表现** | 爆款笔记占比、爆款率 | 中 | 判断内容质量 |
| **粉丝画像** | 年龄、地域、兴趣标签 | 高（平台数据） | 评估受众匹配度 |

### 1.2 单篇笔记维度的数据

| 数据类别 | 具体指标 | 获取难度 | 分析价值 |
|---------|---------|---------|---------|
| **互动数据** | 点赞、收藏、评论、分享数 | 低（公开） | 判断内容表现 |
| **曝光数据** | 浏览量/播放量 | 高（仅博主可见） | 完整效果评估 |
| **内容特征** | 标题、正文、话题标签、封面 | 低（公开） | 爆款要素拆解 |
| **时间特征** | 发布时间、发布日期 | 低（公开） | 最佳发布时机 |
| **流量来源** | 搜索/推荐/同城 | 高（平台数据） | 优化策略 |

### 1.3 行业/趋势维度的数据

| 数据类别 | 具体指标 | 获取难度 | 价值 |
|---------|---------|---------|------|
| **热门话题** | 热搜词、上升话题 | 中 | 抓流量红利 |
| **行业榜单** | 涨粉榜、爆文榜 | 中 | 找对标账号 |
| **季节趋势** | 节日热点、季节需求 | 低 | 提前布局 |

## 二、数据获取路径分析

### 2.1 获取难度 vs 数据价值矩阵

```
高价值 ↑
       │  ┌─────────────┐     ┌─────────────┐
       │  │ 曝光数据     │     │ 粉丝画像    │
       │  │ 完播率       │     │ 流量来源    │
       │  │ (仅博主后台) │     │ (平台数据)  │
       │  └─────────────┘     └─────────────┘
       │
       │  ┌─────────────┐     ┌─────────────┐
       │  │ 涨粉趋势     │     │ 基础画像    │
       │  │ 发布规律     │     │ 互动数据    │
       │  │ (需持续监控) │     │ 内容特征    │
       │  └─────────────┘     └─────────────┘
       │
低价值 └──────────────────────────────────────→ 高难度
       低                                   高
```

### 2.2 各阶段数据获取策略

| 阶段 | 数据来源 | 技术方式 | 可获取数据 |
|------|---------|---------|-----------|
| **MVP** | 用户输入 | Playwright 抓取 | 笔记内容、互动数、博主基础信息 |
| **V2** | 半自动采集 | 定时任务 | 持续监控、历史数据、趋势变化 |
| **V3** | API/数据合作 | 第三方数据源 | 完整数据、粉丝画像、行业数据 |

## 三、AI 分析方法

### 3.1 爆款笔记拆解分析

**输入**：笔记标题 + 正文 + 封面 + 互动数据

**AI 分析输出**：

```
📝 爆款要素拆解报告

📌 标题分析
- 核心关键词：「显瘦穿搭」「梨形身材」
- 情绪触发点：身材焦虑共鸣
- 结构模式：问题 + 解决方案 + 结果

🎨 视觉分析（如有截图）
- 封面风格：清新/高级感/复古
- 色调：低饱和暖色系
- 构图：人物居中 + 纯色背景

💡 内容结构
- 开头：痛点描述（3句话）
- 中间：解决方案（3套穿搭）
- 结尾：互动引导（评论区问问题）

📊 数据表现
- 互动率估算：8.2%（高于同类 2.3 倍）
- 收藏/点赞比：0.6（实用性高）
- 预估曝光：15-20万

🎯 为什么能火？
1. 选题精准：击中梨形身材人群痛点
2. 标题吸睛：数字+具体人群+结果承诺
3. 内容实用：收藏价值高
4. 视觉统一：符合账号调性
```

### 3.2 关键指标计算

**收藏/点赞比**：
```
收藏/点赞比 > 0.5 → 实用性强（干货类）
收藏/点赞比 < 0.2 → 情绪驱动（共鸣类）
```

**互动评分**（参考）：
```
总互动 = 点赞 + 收藏×2 + 分享×3 + 评论×3

> 1000 → 10分
> 500  → 8分
> 100  → 6分
> 50   → 4分
其他    → 2分
```

## 四、机会点分析

### 4.1 竞品工具的盲区

| 竞品盲区 | 我们的机会 |
|---------|-----------|
| **数据堆砌，没有解读** | AI 直接告诉你"该干嘛" |
| **面向品牌/MCN** | 专注 0-1万粉起步博主 |
| **年费数千到数万** | Freemium，低价切入 |
| **功能复杂，学习成本高** | 5分钟上手，极简设计 |
| **只有数据，没有行动** | 周报 + 可执行建议 |
| **不分析"为什么火"** | 爆款要素拆解 |

### 4.2 核心差异化定位

```
传统竞品工具：
数据 → 报表 → 用户自己解读 → 自己想怎么做

WhyFire：
数据 → AI 分析 → 直接告诉你该干嘛 → 为什么这么干
```

## 五、数据风险与合规

### 5.1 法律风险案例

| 时间 | 案例 | 后果 |
|------|------|------|
| 2024年11月 | AI创业团队爬取小红书数据 | 获刑（刑事责任）|
| 2025年4月 | 某公司爬取用户数据转售 | 赔偿 490 万元 |

### 5.2 合规边界

| 行为 | 风险等级 |
|------|---------|
| 用户手动截图上传 | ✅ 低风险 |
| 抓取公开页面信息 | ✅ 低风险 |
| 用户授权后爬取自己的数据 | ⚠️ 中风险 |
| 批量爬取公开数据 | ❌ 高风险 |
| 绕过反爬措施 | ❌ 极高风险 |

### 5.3 WhyFire 的选择

**MVP 阶段采用低风险方案**：
- Playwright 访问公开页面
- 只获取公开可见的信息
- 不需要登录，不需要 Cookie
- 零法律风险
