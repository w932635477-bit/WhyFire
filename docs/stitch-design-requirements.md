# 方言回响 (WhyFire) — UI 设计要求

## 项目概述

**产品名称**: 方言回响 (WhyFire)
**产品定位**: AI 方言翻唱平台。用户上传一首歌，选择方言，30 秒生成方言翻唱，一键生成 MV 和 KTV 歌词视频，可直接发抖音。
**目标用户**: 18-35 岁短视频创作者、方言文化爱好者、抖音/快手用户
**设计风格**: 暗色主题，Apple 产品页面质感，高端极简，沉浸式音乐体验

### 字体

- **字体族**: 系统字体栈 `font-sans`（-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif）
- **主标题**: 32-80px, bold(700), tracking -0.02em ~ -0.03em
- **节标题**: 20-40px, bold(700), tracking -0.02em
- **卡片标题**: 14-20px, semibold(600)
- **正文**: 13-15px, regular(400), line-height 1.6-1.8
- **辅助文字**: 10-12px, regular(400)
- **标签/大写标注**: 10-11px, uppercase, tracking 0.1-0.2em, medium(500)

### 圆角

| 元素 | 圆角 |
|------|------|
| 容器/卡片 | `rounded-2xl` (16px) 或 `rounded-[24px]` |
| 方言卡片 | `rounded-2xl` (16px) |
| 按钮（全宽） | `rounded-full` (999px) |
| 按钮（短） | `rounded-xl` (12px) 或 `rounded-2xl` |
| 输入框 | `rounded-xl` (12px) 或 `rounded-2xl` |
| 播放按钮 | `rounded-full` (圆形) |
| 作品封面 | `rounded-[20px]` |
| 步骤指示器 | `rounded-full` (圆形) |
| 导航栏 | `rounded-b-2xl` (仅底部圆角) |

### 间距

- 页面内容最大宽度: `640px`（翻唱流程）/ `980px`（首页）
- 水平内边距: `24px` (px-6)
- 节间距: `80-112px` (py-20 ~ py-28)
- 卡片内边距: `24-32px` (p-6 ~ p-8)
- 元素间距: `space-y-6` ~ `space-y-12`

### 动效

- **按钮按压缩放**: `active:scale-[0.98]`
- **卡片 hover**: `transition-all duration-300` ~ `duration-500`
- **淡入动画**: `animate-fade-in`
- **旋转加载**: `animate-spin`（2s 周期）
- **渐变进度条**: `transition-all duration-1000 ease-out`
- **hover 缩放**: `hover:scale-[1.02]` 或 `hover:scale-105`

## 页面规格

### 页面 1: 首页 `/sonic-gallery`

**功能**: 产品介绍、热门作品展示、方言展示、功能亮点、引导用户进入翻唱

**布局**: 单列居中，最大宽度 980px

**区域划分**:

1. **导航栏** — 固定顶部，毛玻璃效果
   - 背景: `rgba(0,0,0,0.7)` + `backdrop-blur-2xl`
   - 左侧: Logo（白色 W 方块 + "方言回响"文字）
   - 右侧: 三个导航链接（翻唱 / Rap / 作品），11px 白色半透明
   - 底部圆角: `rounded-b-2xl`

2. **Hero 区域** — 全宽，背景视频 + 暗色遮罩
   - 高度: 视口 40-50%
   - 背景: 自动播放静音视频，30% 透明度 + 从上到下渐变遮罩
   - 内容: 居中排列
     - 上方标签: "全新上线"，11px 大写，tracking 0.2em，白色 30%
     - 主标题: 48-80px 粗体，"用方言翻唱" + "30 秒出片"（渐变文字）
     - 副标题: 17-19px，白色 50%，描述性文字
     - 双按钮: 主按钮（白底黑字"方言翻唱"）+ 次按钮（边框"原创 Rap"），均为 pill 形状
     - 数据行: 三组数字展示（9 种方言 / 30s 出翻唱 / MV 视频直出）

3. **热门作品** — 水平网格
   - 标题 + "查看全部"链接
   - 4 列网格（移动端 2 列），间距 20px
   - 每个作品卡片:
     - 正方形封面（圆角 20px），渐变色背景
     - 中央播放按钮（半透明黑圆 + 白色三角）
     - 底部方言标签 + 播放量
     - hover 时整体放大 2%
     - 卡片下方: 标题（14px 粗体）+ 创作者名（12px 半透明）

4. **方言展示** — 居中 flex wrap 布局
   - 标题: "9 种方言" + 副标题
   - 每个方言: 96px 宽的方形卡片
     - 上方: 方言首字，圆形背景（对应色 15% 透明度）
     - 下方: 方言名 + 地区名
     - hover 时首字放大 10%

5. **功能亮点** — 3 列卡片网格（移动端单列）
   - 每张卡片: 深灰背景，24px 圆角
   - 内容: 图标（紫色线条）+ 标题 + 描述 + 底部分割线 + 大数字统计
   - hover 时背景变浅灰

6. **底部 CTA** — 居中排列
   - 大标题 "准备好了吗？" + 描述 + 白色 CTA 按钮

7. **页脚** — 极简
   - Logo + 版权 + 隐私/条款/联系链接

**组件**:

1. **顶部导航** — 固定，毛玻璃
   - 左侧: 返回箭头
   - 中间: 步骤指示器（3 个圆点，当前步骤白色实心，已完成绿色勾号，未开始灰色）
   - 右侧: W Logo

2. **标题区** — "上传原唱" (32px 粗体) + 描述

3. **上传区域** — 大卡片（圆角 24px，内边距 64px）
   - 状态一（空）: 上传图标 + "拖拽音频或视频文件到这里，或点击选择" + 支持格式说明
   - 状态二（拖拽中）: 紫色边框 + 微放大
   - 状态三（上传中）: 旋转加载圈 + "正在上传..."

7. **预览播放器** — 上传完成后显示

   - 进度条（2px 高，白色，可点击跳转）
   - 时间显示（两端，10px）

8. **下一步按钮** — 白色全宽 pill 按钮，右箭头图标

9. **底部导航栏** — 固定底部，毛玻璃，"上一步" + "下一步"

### 页面 3: 翻唱流程 - 步骤 2 方言与歌词 (Step 2)

**功能**: 选择方言、歌词模式、人声性别

**组件**:

1. **方言选择** — 网格布局
   - 标题: "选方言" (32px 粗体) + "决定翻唱的味道"
   - 3x3 网格（桌面 5 列），间距 12px
   - 每个方言按钮:
     - 深灰背景，圆角 16px
     - 方言名（13px 粗体）+ 地区（10px 半透明）
     - 选中态: 背景变浅灰 + 右上角方言色圆点（直径 8px）
     - hover: 背景微亮

2. **歌词模式** — Apple 分段控制器
   - 标题: "歌词" (20px) + 描述
   - 分段控制器: 深灰背景容器，三个等宽按钮（保留原词 / 自定义 / AI 创作）
   - 选中态: 浅灰背景 + 白色文字
   - 未选中: 白色 25%

3. **自定义歌词输入** — mode=custom 时显示
   - 大文本框，8 行，16px 圆角

4. **AI 歌词生成** — mode=ai-generate 时显示

   - 生成结果可编辑文本框（紫色边框提示）

5. **人声选择** — 两列按钮
   - "男声" / "女声"，等宽
   - 选中: 浅灰背景白色文字

6. **下一步按钮** — 白色全宽 pill

### 页面 4: 翻唱流程 - 步骤 3 预览与生成 (Step 3)

**功能**: 确认生成、等待翻唱、播放结果、生成 MV、生成 KTV 歌词视频

这是最复杂的页面，有 4 种主要状态:

**状态 A: 确认页（初始）**

1. **标题**: "确认生成" (32px) + 描述
2. **设置概览卡片** — 2 列网格
   - 方言 + 人声性别
   - 歌词模式（底部全宽）
3. **"开始翻唱" 按钮** — 白色全宽 pill，星形图标
4. **预计时间提示** — 极淡文字 "预计 30 秒"

**状态 B: 生成中**

1. **旋转音符动画** — 多层圆环 + 中央音符 SVG
   - 外圈: 细白色边框
   - 中圈: 白色顶部弧段，2 秒旋转动画
   - 内部: 音符图标
2. **状态文案** — 方言名 + "翻唱"，20px 粗体
   - 副标题: 进度消息（14px，半透明白色）
3. **进度条** — 56px 宽
   - 底: 白色 6%
   - 填充: 紫→绿渐变，`ease-out` 过渡
   - 百分比数字: 右对齐，11px
4. **趣味提示** — 每 5 秒切换一条，淡入动画

**状态 C: 完成**

1. **成功提示** — 绿色背景卡片
   - 绿色圆形勾号 + 方言名 + "翻唱完成" + 时长
2. **音频播放器** — 深灰卡片
   - 16px 白色圆形播放按钮
   - 歌曲名 + "方言回响"
   - **波形可视化**: 60 根细条（最小宽度 1.5px），已播放白色，未播放 10% 白色
   - 点击波形跳转播放位置
   - 时间显示（两端）
3. **歌词展示** — 深灰卡片
   - 标题 "歌词" + 字数
   - 歌词文本，13px，最大高度 240px 可滚动
4. **MV 视频** — 深灰卡片
   - 标题 "MV 视频" + "可直接发抖音"
   - 空闲态: "生成音乐视频" 按钮（带摄像图标）
   - 生成中: 旋转加载 + 进度条 + "预计 1-3 分钟"
   - 完成态: `<video>` 播放器，圆角，最大高度 360px
   - 失败态: 错误提示 + 重试链接
5. **KTV 歌词视频** — 深/渐变边框卡片（仅在 MV 完成后显示）
   - 标题 "KTV 歌词视频" + "逐字高亮"
   - 空闲态: 渐变边框按钮 "生成 KTV 歌词视频"（麦克风图标）
   - 生成中: 紫色旋转加载 + 紫色进度条 + "FFmpeg 客户端渲染，无需服务器"
   - 完成态: 视频播放器 + 紫色下载按钮
   - 失败态: 错误提示 + 重试
6. **操作按钮** — 两列
   - 下载（白色 pill）+ 分享（深灰 pill）

**状态 D: 失败**

1. **红色圆形 X 图标**
2. **"翻唱生成失败"** (20px)
3. **错误信息** (13px 半透明)
4. **重试按钮** — 白色 pill

## 交互模式

### 步骤导航

- 3 步线性流程: 上传 → 方言/歌词 → 生成
- 步骤指示器在顶部，可点击已完成的步骤跳回
- 底部固定导航栏（步骤 1-2 之间显示）
- 步骤 3 生成中不可后退

### 轮询机制

- 翻唱生成: 3 秒间隔轮询，带真实进度（0-100%）
- MV 生成: 3 秒间隔轮询
- KTV 生成: 客户端 FFmpeg 渲染，实时进度回调

### 按钮状态

- 主按钮（白底黑字）: 默认态可用，按下缩放 0.98
- 次按钮（深灰底）: hover 变浅灰
- 禁用态: 10-25% 透明度，cursor-not-allowed
- 加载态: 旋转圈 + 文字

### 表单交互

- 输入框: 深灰背景，白色 6% 边框，focus 时 15% 边框
- 文本域: 圆角 16px，8 行高度
- 分段控制器: Apple 风格，滑块动画

| 断点 | 宽度 | 变化 |
|------|------|------|
| 移动端 | < 640px | 单列布局，方言网格 3 列，热门作品 2 列，隐藏步骤文字标签 |
| 平板 | 640-768px | 方言网格 5 列，热门作品 2-4 列 |
| 桌面 | > 768px | 热门作品 4 列，功能亮点 3 列，显示步骤文字标签 |

## 关键设计原则

1. **暗色沉浸感**: 纯黑背景，所有卡片、按钮、文字都在深灰色阶中展开
2. **Apple 质感**: 大标题、紧凑 tracking、pill 按钮、毛玻璃导航栏
3. **渐变品牌色**: 紫(#8b5cf6)→绿(#10b981) 贯穿所有进度条、CTA 装饰、标题渐变
4. **低信息密度**: 每屏只做一件事，大量留白
5. **方言个性**: 每个方言有独特颜色标识，让选择有仪式感
6. **全宽 CTA**: 所有主要操作都是全宽 pill 按钮，移动端友好
7. **真实进度反馈**: 生成过程中显示真实进度百分比和阶段描述
8. **渐进式功能展示**: MV 生成完成后才出现 KTV 入口，避免信息过载

## 技术约束

- **框架**: Next.js 15 + React + Tailwind CSS
- **主题**: 纯暗色（无亮色模式切换）
- **动画**: CSS transitions + Tailwind 动画类（无额外动画库）
- **图标**: 内联 SVG，线条风格，strokeWidth 1.5-2.5
- **视频播放器**: 原生 `<video>` 标签，无第三方播放器
- **音频播放器**: 原生 `<audio>` 标签 + 自定义 UI
- **最大内容宽度**: 翻唱流程 640px，首页 980px
